# 六爻排盘数据源集成调研

## 目标

- **输入**：六次摇卦结果，每次为 `6 | 7 | 8 | 9`（老阴/少阳/少阴/老阳）
- **输出**：完整排盘 JSON，包含：本卦、变卦、卦宫、纳甲、六亲、世应、六神、动爻
- **对接**：将排盘结果接到当前前端 UI（结果页展示）

- 铜钱和：6 → 老阴，7 → 少阳，8 → 少阴，9 → 老阳

## 2. 最适合集成的方案

**推荐：优先集成 najia-core**

- 与现有 **TypeScript / Next.js** 技术栈一致，无需额外运行时或 Python 服务。
- 输入输出清晰：由当前 `diceSums: number[]` 转成 `HexagramData`，调用 `getNaJiaData`、`getTransformedHexagram` 即可得到纳甲、六亲、六神、世应、变卦；**卦宫**可通过库内八宫规则自建一小段映射（或后续向库提 PR 暴露）。
- 无外部请求与计费，延迟可控，便于在 `/api/analyze` 或单独 `/api/liuyao-board` 中直接调用。

**备选**：

- **缘份居**：若需要「四柱、卦辞、事业/婚恋等现成文案」且可接受付费，可作为补充或替代。
- **bopo/najia**：仅在必须使用 Python 生态或需其特有规则时再考虑，并需增加一层 Python 服务或 Serverless。

## 3. 接入步骤（以 najia-core 为例）

1. **安装依赖**（已完成）  
   `npm install najia-core`

2. **封装「6/7/8/9 → 排盘」**  
   - 将 `diceSums`（初爻→六爻顺序）转为 `HexagramData.lines`（YaoType[]）。  
   - 调用 `getNaJiaData(hexagram, date)` 得到 6 条 NaJiaLine。  
   - 调用 `getTransformedHexagram(hexagram)` 得到变卦；若有动爻，可再对变卦调用 `getNaJiaData`（或按需只取变卦卦象）。  
   - 动爻：从 `hexagram.lines` 中 `old_yin` / `old_yang` 的索引得到。  
   - 卦宫：najia-core 内部用 `palaceName`（如 qian/kun），可自建 `palaceName → "乾宫"/"坤宫"/...` 的映射表（见下方数据结构）。

3. **定义统一排盘 JSON 结构**  
   在 `types/divination.ts` 或新建 `types/liuyao-board.ts` 中定义 `LiuYaoBoard`（见第 4 节），供 API 与前端共用。

4. **API 层**  
   - 方案 A：在现有 `POST /api/analyze` 中，在 `computeDivinationResult(diceSums)` 之后调用排盘封装，将 `board: LiuYaoBoard` 一并写入 session。  
   - 方案 B：新增 `POST /api/liuyao-board`，入参 `{ diceSums: number[], date?: string }`，返回 `LiuYaoBoard`，前端在结果页或摇卦后单独请求。  
   推荐先采用 **方案 A**，保证一次请求得到「基础卦象 + 完整排盘 + AI 分析」。

5. **存储**  
   在 `StoredDivinationSession` 中增加可选字段 `board?: LiuYaoBoard`；`saveSession` 时若已计算则写入。

6. **前端**  
   结果页从 `session.board` 读取，用表格或现有 `ProfessionalHexagramTable` 扩展展示：卦宫、纳甲、六亲、世应、六神、动爻等。

## 4. 请求 / 返回数据结构

### 4.1 当前项目（输入，已存在）

- **摇卦结果**：`diceSums: number[]`，长度为 6，每个元素为 6 | 7 | 8 | 9（初爻到上爻）。
- 可选：`date?: string`（ISO 或 YYYY-MM-DD），用于排六神、四柱；不传则用服务器当前时间。

### 4.2 目标排盘 JSON（输出，建议结构）

```ts
// types/liuyao-board.ts（建议）

/** 单爻排盘信息 */
export interface LiuYaoLineBoard {
  /** 爻位 1-6，1 为初爻 */
  index: number;
  /** 纳甲：天干+地支，如 "壬戌" */
  naJia: string;
  /** 五行 */
  fiveElement: string;
  /** 六亲 */
  liuQin: string;
  /** 六神 */
  sixGod: string;
  /** 世应：'世' | '应' | undefined */
  shiYing?: string;
  /** 是否动爻 */
  moving: boolean;
  /** 伏神（若有） */
  fuShen?: { naJia: string; fiveElement: string; liuQin: string };
}

/** 卦宫名 */
export type PalaceName =
  | "乾宫" | "坤宫" | "震宫" | "巽宫" | "坎宫" | "离宫" | "艮宫" | "兑宫";

export interface LiuYaoBoard {
  /** 本卦卦名，如 "山水蒙" */
  benGuaName: string;
  /** 本卦卦宫 */
  benGuaPalace: PalaceName;
  /** 本卦六爻（初爻→上爻） */
  benGuaLines: LiuYaoLineBoard[];
  /** 动爻爻位列表，如 [1, 5] */
  movingPositions: number[];
  /** 变卦卦名（无动爻时可无） */
  bianGuaName?: string;
  /** 变卦卦宫（无动爻时可无） */
  bianGuaPalace?: PalaceName;
  /** 变卦六爻（无动爻时可无） */
  bianGuaLines?: LiuYaoLineBoard[];
  /** 世爻所在爻位 1-6 */
  shiPosition: number;
  /** 应爻所在爻位 1-6 */
  yingPosition: number;
  /** 四柱/日空等可选：可由 astrology 或缘份居补充 */
  // siZhu?: { year: string; month: string; day: string; hour: string };
  // riKong?: string;
}
```

### 4.3 缘份居 API（若采用商业接口）

- **请求**：`POST https://api.yuanfenju.com/index.php/v1/Liupan/liuyao`  
  - `Content-Type: application/x-www-form-urlencoded`  
  - 手工指定：`pan_model=4`，`gua_yao1`～`gua_yao6`（0/1/2/3），`sex`、`born_year` 等。  
  - 编码映射：6→2, 7→1, 8→0, 9→3（初爻对应 gua_yao1，六爻对应 gua_yao6）。

- **返回**：`errcode`、`errmsg`、`data`。  
  - `data.gua_info.bengua`：本卦卦名、卦宫、卦爻信息（六亲、世应、六神、伏神等）。  
  - `data.gua_info.biangua`：变卦（当 `has_biangua === "1"`）。  
  - `data.dongyao`：动爻位，如 `"1"` 或 `"5,6"`。  
  - 详见 [缘份居六爻排盘文档](https://doc.yuanfenju.com/liupan/liuyao.html)。

## 5. najia-core 最小可运行 Demo

### 方式一：Next.js API（推荐，可直接验证）

项目内已提供 **`app/api/demo-liuyao-board/route.ts`**，在 Next 环境中运行排盘逻辑并返回 JSON。

```bash
npm run dev
# 浏览器或 curl 请求
curl http://localhost:3000/api/demo-liuyao-board
```

返回示例：`input.diceSums`、`benGua.lines`（纳甲/六亲/六神/世应/动爻）、`benGua.movingPositions`、`bianGua.lines`（变卦）、`siZhu`（四柱/日空）等。

### 方式二：脚本逻辑参考

**`scripts/najia-board-demo.mjs`** 为同一逻辑的脚本形式。因 najia-core 的 ESM 在 Node 下存在无扩展名导入问题，**直接用 `node scripts/najia-board-demo.mjs` 可能报错**，仅作阅读/复制用；实际可运行 demo 以方式一为准。

Demo 逻辑简述：

1. 用 6/7/8/9 数组构造 `HexagramData`（6→old_yin, 7→young_yang, 8→young_yin, 9→old_yang）。
2. 调用 `getNaJiaData(hexagram, new Date())`，得到 6 条纳甲/六亲/六神/世应。
3. 调用 `getTransformedHexagram(hexagram)` 得到变卦；若有动爻，再对变卦调用 `getNaJiaData` 得到变卦六亲等。
4. 动爻：遍历 `hexagram.lines`，收集 `old_yin`/`old_yang` 的索引（1-based）。
5. 卦宫：najia-core 未导出，demo 中用简单映射表（palaceName 英文 → 中文宫名）占位；正式接入时可从库内 `getHexagramInfo` 逻辑抽取或提 PR 暴露。
6. 本卦卦名：可用现有 `lib/gua-data.ts` 的 `getHexagramMetaByCode(binaryCode)` 得到（需保证 binaryCode 与 najia 一致：初爻为 bits[0]，上爻为 bits[5]，1=阳 0=阴）。

以上完成后，即可在正式 API（如 `/api/analyze`）中复用 demo 中的「diceSums → HexagramData → getNaJiaData + getTransformedHexagram」流程，输出统一 `LiuYaoBoard` 并接到前端。
